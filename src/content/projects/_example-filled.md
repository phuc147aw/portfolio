## Context

2–4 sentences setting the scene. What is this project? What environment or need does it address? Help someone unfamiliar with the domain understand the purpose.

## Problem

What specific challenge or gap does this project solve? Be concrete: mention pain points, limitations of existing solutions, or operational needs that motivated the work.

- Bullet points work well here
- Focus on measurable problems when possible
- Explain why existing solutions weren't sufficient

## Solution

I designed and built **Example Project** to solve this:

- **Feature one** — brief explanation of what it does and why
- **Feature two** — architecture decision and its benefit
- **Feature three** — technical approach that makes this unique
- Deployed with Docker for simple self-hosting

Key decisions:
- Chose technology X over Y because of Z
- Built a plugin architecture for extensibility
- Used approach A for performance reasons

## Outcome

- 50% reduction in manual effort for the target task
- Adopted by N users/teams within the first quarter
- Open-sourced with documentation — community contributions followed
- Architecture pattern reused in subsequent projects

## Tech Stack

React, TypeScript, Node.js, PostgreSQL, Docker
